# Security Policy

We take the security of **Glitch Operation (Bug Bounty Platform)** very seriously. If you believe you have found a security vulnerability in this project, we encourage you to let us know right away.

---

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.** Instead, report them privately via email:

*   **Email**: [ops.glitch.hack@gmail.com](mailto:ops.glitch.hack@gmail.com)

When reporting a vulnerability, please include:
1. A detailed description of the vulnerability.
2. Step-by-step instructions (with screenshots or proof of concept) to reproduce it.
3. The potential impact of the vulnerability.

We will acknowledge receipt of your vulnerability report within **48 hours** and provide periodic updates as we investigate and resolve the issue.

---

## Safe Harbor & Responsible Disclosure

We expect contributors and security researchers to adhere to responsible disclosure principles:
*   Give us reasonable time to investigate and fix the issue before making any public disclosures.
*   Do not exploit or access data belonging to other users without authorization.
*   Do not use automated scanners or perform DDoS attacks against public staging or production sites.
*   Acts of good faith in reporting vulnerabilities according to these guidelines will not be subject to legal action.
